# 프로젝트 개요: Quiz Battle Card (실시간 웹 게임)
- **타겟:** 초등학생 (18~25명 개인전 진행)
- **기술 스택:** 바닐라 HTML, CSS, JavaScript (프레임워크 미사용)
- **데이터베이스:** Firebase Realtime Database (실시간 동기화) — **qbr과 별도의 새 Firebase 프로젝트 사용**
- **문제 파싱:** .xlsx 파일 업로드 → FileReader API로 파일을 읽고 → SheetJS(CDN) 라이브러리로 표 데이터로 변환 (qbr과 동일)
- **구조:** SPA (Single Page Application). 팝업(Modal) 절대 금지, 모든 화면 이동은 부드러운 '전체 화면 전환(Full-page transition)'으로 구현.

> **qbr(Quiz Battle Royale)과의 핵심 차이:** 문제를 맞힌 뒤 **카드를 선택**하고, 그 카드의 점수를 획득하여 합산한다. 순위는 정답 순서가 아닌 **누적 점수**로 매긴다.

# 1. 화면 구성 (총 4개 View)

## 1-1. Setting View (세팅)
- **플레이어 수 선택 없음** (qbr과 달리 드롭다운 제거). 닉네임 입장 순서대로 참여.
- 방 코드 직접 입력 필드: **알파벳+숫자 4자리 조합** (예: AB3K). Admin과 클라이언트가 동일한 코드를 입력하여 같은 방에 입장.
- .xlsx 파일 업로드 버튼 (문제 데이터 로드). 업로드 완료 시 문제 수 및 미리보기 표시.
- '게임준비' 버튼

## 1-2. Board View (게임상황판) — 교실 스크린용
- **좌측 40%:** 문제 내용 및 이미지 표시. 이미지 클릭 시 전체화면 일시 확대. (qbr은 상단 40%였으나 qbc는 좌측 40%)
- **우측 60%:** 뒤집힌 카드를 **가로 7장 × 세로 4장, 총 28장** 배열.
  - 클라이언트가 선택(뒤집은) 카드는 실시간 동기화되어 board에서도 뒤집힘.
  - **카드가 뒤집히면 해당 카드의 점수/내용을 공개**하고, **그 카드를 고른 플레이어의 닉네임을 함께 표시**한다.
  - **카드 뒷면이 등급별 색상으로 구분된다:** 일반=노란색, 위험=주황색, 고위험=빨간색, 초고위험=검정색. 학생은 **뒷면 색깔을 보고 안정적인 카드를 고를지 위험을 감수할지 선택**한다. (이것이 게임의 핵심 선택 요소이므로 뒷면 색상은 반드시 등급을 드러내야 한다.)
  - **뒤집으면 점수/내용이 공개**된다. 즉 색깔 = 등급(위험도)은 처음부터 보이고, 구체적 점수는 뒤집어야 안다.
  - **대폭발 발동 시:** 별도의 연출(효과음+애니메이션)을 표시하되, **점수는 보여주지 않는다.**
  - **주의:** 카드 개별 점수는 공개하되, **플레이어별 누적 점수는 게임이 끝날 때까지 공개하지 않는다.**

## 1-3. Client View (클라이언트) — 학생 접속용
- 방 코드 입력 후 **닉네임 입력** (qbr의 P1~P6 고정 번호 방식 대신 닉네임 사용).
- 문제 유형별 정답 입력 UI 표시.
- **제출 후 UI 잠금:** 한 번 제출하면 재시도 불가, 입력 UI 비활성화.
- **정답 판정 후 카드 선택 화면으로 전환:**
  - **정답을 맞힌 경우:** 카드 선택 화면으로 전환되어 28장 중 하나를 클릭해 선택 가능.
  - **오답인 경우:** 카드 선택 불가. board와 동일하게 다른 플레이어가 카드를 뒤집는 모습을 구경만 함 (점수 획득 없음).
  - 다른 클라이언트가 선택하는 카드가 동기화되어 내 화면에서도 뒤집힘 (소진 표시).
  - **내가 선택한 카드는 크게 확대하여 3초간 보여줌** (점수/내용 확인).

## 1-4. Admin View (관리자) — 교사용 통제 패널
- 다음 문제로 수동 이동 버튼 (관리자가 직접 눌러야만 넘어감)
- 문제 건너뛰기 버튼
- 현재 문제 번호 / 전체 문제 수 표시
- 5초 카운트다운 트리거 버튼 (모든 화면에 카운트다운 동기화됨)
- 현재 플레이어별 제출 현황 표시
- '게임 끝내기' 버튼 → 축포 애니메이션 + 효과음 + **최종 순위(누적 점수) 표시**

### qbc 추가 기능
- **'초고위험카드' 버튼:** 누르면 `gameState/ultraArmed = true`. **카운트다운 시작 전에** 눌러야 하며, 카운트다운 시작 시 카드 배분에 반영된 뒤 **자동으로 false로 리셋**된다(1회성).
- **'대폭발 포함' 토글:** `gameState/explosionEnabled`를 켜고 끈다. 꺼져 있으면 초고위험카드 점수 풀에서 대폭발이 제외된다.
- **(이미 구현됨) 모든 카드 내용 보기:** 관리자만 28장의 등급·점수를 미리 볼 수 있다. 이 정보는 board/client에 절대 노출되지 않아야 한다.
- **(이미 구현됨) 모든 카드 뒤집기 버튼:** 문제 종료 후 전체 카드를 공개하는 용도. 단, 기본 규칙상 **아무도 안 고른 카드는 자동 공개되지 않으므로**, 이 버튼은 관리자가 의도적으로 누를 때만 동작한다.

# 2. 카드 시스템 (qbc 핵심 엔진)

## 2-1. 카드 등급 및 배분
- 매 문제마다 **28장(7×4)의 카드를 서버에서 새로 랜덤 배분**한다. 카드가 소진되어도 다음 문제에서 28장이 새로 채워지므로 인원 대비 카드 부족 문제는 발생하지 않는다.
- **카드 등급별 장수 및 색상:**

| 등급 | 색상 | 장수 | 점수 풀 |
|---|---|---|---|
| 일반카드 | 노란색 | 19장 | 5, 6, 7, 8, 9 |
| 위험카드 | 주황색 | 5장 | 0, 1, 2, 3, 9, 10, 11 |
| 고위험카드 | 빨간색 | 4장 | -20, -15, -10, -5, 0, 10, 20, **2배카드** |
| 초고위험카드 | 검정색 | (0 또는 1장) | -40, -35, -30, -25, -20, 25, 30, 40, **대폭발** |

- **점수 풀의 중복은 의도된 설계다.** (위험카드의 9~11점이 일반카드보다 높고, 0점이 여러 등급에 존재하는 것 등) 임의로 정리하지 말 것.
- **평상시:** 일반 19 + 위험 5 + 고위험 4 = **28장**
- **초고위험 모드:** 일반 19 + 위험 5 + 고위험 **3** + 초고위험 **1** = **28장** (고위험 1장을 초고위험으로 대체)
- 배분 시점: **카운트다운 시작 시 관리자 측에서 28장을 생성해 Firebase에 저장** → 모든 화면이 동일한 카드 세트를 참조.

## 2-1-1. 초고위험카드 (관리자 트리거)
- 관리자 페이지의 **'초고위험카드' 버튼**을 누른 문제에만 등장한다.
- **1회성:** 버튼을 누르면 **그 문제 한 번만** 적용되고, 다음 문제에는 자동으로 해제된다. (토글이 아님)
- **누르는 시점:** 카드 배분이 카운트다운 시작 시점에 일어나므로, **카운트다운 시작 전에** 눌러야 반영된다. 카운트다운 시작 후에는 반영되지 않는다.
- **별도 예고 표시 없음:** board/client에 "이번 문제에 초고위험이 있다"는 안내 문구·배너는 띄우지 않는다. 단, **뒷면이 검정색이므로 카드판을 보면 초고위험 존재 자체는 드러난다.** (이는 뒷면 색상 규칙상 의도된 결과 — 학생은 검정 카드를 피할지 노릴지 선택할 수 있다.)
- **아무도 안 고르면 아무 일도 일어나지 않는다.** 문제 종료 시 남은 카드를 공개하지 않는다.

## 2-1-2. 대폭발 카드
- 효과: **모든 플레이어의 누적 점수가 0이 된다.** (뽑은 본인 포함)
- 초고위험카드 점수 풀에 포함되며, **weight로 확률이 결정**된다. 즉 초고위험카드가 등장해도 대폭발이 아닐 수 있다.
- **관리자가 대폭발 포함 여부를 별도로 켜고 끌 수 있어야 한다.** (`explosionEnabled` 설정) 꺼져 있으면 초고위험카드 점수 풀에서 대폭발을 제외하고 뽑는다.

## 2-1-3. 카드 설정값 분리 (수정 용이성)
- 각 등급의 **점수 풀**과 **등장 확률(가중치)**, **장수**는 나중에 쉽게 바꿀 수 있도록 **별도의 설정 객체(config)로 분리**한다. 배분 로직 코드 안에 숫자를 하드코딩하지 않는다.
- 예시 구조 (weight 값은 임의값, 실제 밸런스는 조정 필요):

```javascript
const CARD_CONFIG = {
  // 등급별 장수
  counts: {
    normal: 19,   // 노란색
    risk: 5,      // 주황색
    highRisk: 4,  // 빨간색 (초고위험 모드에서는 3장으로 줄고 ultra 1장 추가)
  },

  // 등급별 색상 (UI에서 참조)
  colors: {
    normal:   "#FFD700",  // 노란색
    risk:     "#FF8C00",  // 주황색
    highRisk: "#DC143C",  // 빨간색
    ultra:    "#1A1A1A",  // 검정색
  },

  // 등급별 점수 풀과 가중치
  tiers: {
    normal: [
      { score: 5, weight: 1 },
      { score: 6, weight: 1 },
      { score: 7, weight: 1 },
      { score: 8, weight: 1 },
      { score: 9, weight: 1 },
    ],
    risk: [
      { score: 0,  weight: 1 },
      { score: 1,  weight: 1 },
      { score: 2,  weight: 1 },
      { score: 3,  weight: 1 },
      { score: 9,  weight: 1 },
      { score: 10, weight: 1 },
      { score: 11, weight: 1 },
    ],
    highRisk: [
      { score: -20,      weight: 1 },
      { score: -15,      weight: 1 },
      { score: -10,      weight: 1 },
      { score: -5,       weight: 1 },
      { score: 0,        weight: 1 },
      { score: 10,       weight: 1 },
      { score: 20,       weight: 1 },
      { score: "double", weight: 1, max: 1 },  // 세트당 최대 1장
    ],
    ultra: [
      { score: -40,         weight: 1 },
      { score: -35,         weight: 1 },
      { score: -30,         weight: 1 },
      { score: -25,         weight: 1 },
      { score: -20,         weight: 1 },
      { score: 25,          weight: 1 },
      { score: 30,          weight: 1 },
      { score: 40,          weight: 1 },
      { score: "explosion", weight: 1 },  // 대폭발: explosionEnabled가 false면 제외
    ],
  },

  explosionEnabled: true,  // 관리자가 켜고 끔
};
```

- 배분 알고리즘:
  1. `counts.normal`만큼 `tiers.normal` 가중치로 뽑기
  2. `counts.risk`만큼 `tiers.risk` 가중치로 뽑기
  3. 초고위험 모드가 **아니면** `counts.highRisk`(4장)만큼 `tiers.highRisk`로 뽑기
  4. 초고위험 모드**이면** `counts.highRisk - 1`(3장)만큼 `tiers.highRisk`로 뽑고, `tiers.ultra`에서 1장 뽑기 (`explosionEnabled`가 false면 explosion 항목 제외 후 뽑기)
  5. `double`은 `max: 1` 제한 준수
  6. 뽑힌 값이 `"double"`/`"explosion"`이면 카드의 `type`을 그 값으로, `score`는 0으로 저장한다 (기존 코드 관례 유지 — 2배카드·대폭발 판정이 `type` 기준으로 동작하므로 `tier`가 아닌 `type` 필드를 그대로 씀). 그 외 카드는 `type`에 등급명(normal/risk/highRisk/ultra), `score`에 뽑힌 숫자를 저장한다.
  7. 28장을 합쳐 무작위로 섞어 `cards/N/deck`에 저장
- 이렇게 하면 나중에 점수·확률·장수를 바꿀 때 `CARD_CONFIG` 값만 수정하면 되고, 배분 로직은 손대지 않아도 된다.

## 2-2. 카드 선택 (소진형)
- 카드 선택 순서: **정답을 빨리 맞힌 사람부터** 선택.
- **소진형:** 한 번 선택(뒤집힌) 카드는 공개되며, 다른 플레이어는 그 카드를 선택할 수 없다.
- **동시 클릭 충돌 처리:** 두 명이 거의 동시에 같은 카드를 탭할 수 있으므로, **Firebase 트랜잭션**으로 "먼저 카드를 확정한 사람만 성공" 처리한다. 실패한 플레이어는 다른(아직 뒤집히지 않은) 카드를 골라야 한다.

## 2-3. '다음카드 2배' 카드 (고위험/빨간색)
- **2배카드 자체의 점수는 0점.**
- 2배카드를 선택하면 **카드를 한 장 더 고르도록** 하고, 그 **두 번째 카드의 점수를 2배**로 적용한다.
- 두 번째 카드 선택도 **소진형** — 남들이 이미 뒤집지 않은 카드 중에서 고른다.
- **부호 무관하게 2배:** 두 번째 카드가 마이너스여도 2배 적용 (예: `-20` × 2 = `-40`).
- 세트당 2배카드가 1장뿐이므로 두 번째 카드가 또 2배카드일 수는 없다.
- **두 번째 카드가 초고위험카드일 수 있다.** 이 경우도 2배 적용 (예: `-40` × 2 = `-80`, `40` × 2 = `80`). 두 번째 카드가 **대폭발**이면 2배와 무관하게 대폭발 효과가 발동한다.

## 2-4. '대폭발' 카드 (초고위험/검정색)
- 효과: **모든 플레이어의 `totalScore`를 0으로 초기화한다.** 뽑은 본인도 포함.
- 초기화 시점: 카드를 뽑아 확정된 즉시.
- 대폭발이 발동해도 **누적 점수는 여전히 비공개**이므로, board에는 "대폭발 발동" 연출만 표시하고 점수는 보여주지 않는다.
- 대폭발 카드를 뽑은 플레이어의 `gainedScore`는 0으로 기록하고, 별도 플래그(`explosionTriggeredBy`)로 발동 사실을 남긴다.

# 3. 순위 판정
- **정답 순서가 아닌 누적 점수로 순위를 매긴다** (qbr과의 핵심 차이).
- 각 문제에서 정답자가 획득한 카드 점수를 합산 → 누적 점수 기준 내림차순 순위.
- 오답자는 해당 문제에서 카드 선택 불가 → 점수 획득 없음 (0점 반영).
- **점수·순위는 게임 종료 시점('게임 끝내기' 클릭)에만 공개.** 매 문제 후에는 비공개.

# 4. 문제 유형별 판정

> 엑셀 데이터 형태는 qbr과 동일. 유형 값: `객관식`, `단답형`, `선잇기`, `복수정답`, `순서`.

## 4-1. 객관식
- **qbr의 난수(10~99) 부여 기능 제거.** 보기를 있는 그대로 표시하고, 정답 판정을 단순화한다.
- 클라이언트는 보기 중 하나를 선택해 제출. 선택한 보기가 정답 데이터와 일치하면 정답.

## 4-2. 단답형 (qbr과 동일)
- 타이핑 후 '제출' 버튼 클릭.
- **입력값과 정답 데이터의 모든 공백을 제거(`replace(/\s+/g, '')`)하고 비교.**
- 복수 정답 허용 (정답 데이터에 `|`로 구분).

## 4-3. 선 잇기 (매칭) (qbr과 동일)
- HTML5 Canvas 사용.
- 정답 데이터는 `1:one | 2:two | 3:three` 형태. 파이프(`|`)와 콜론(`:`)으로 파싱하여 좌우 무작위 배치.
- 캔버스 CSS에 `touch-action: none;` 반드시 적용.
- 반응형 좌표 계산 필수. 모든 항목 연결 시에만 '제출' 활성화.

## 4-4. 복수정답 (qbr과 동일)
- 정답 데이터는 `1,4` 형태 (쉼표 구분).
- 보기를 있는 그대로 표시. 해당 항목 모두 체크 후 '제출'.

## 4-5. 순서 (qbr과 동일)
- 정답 데이터는 `1,4,3,2` 형태 (올바른 순서).
- 보기를 순서대로 클릭 선택. 이미 선택한 항목 재클릭 시 그 이후 선택 전체 취소.
- 올바른 순서로 모두 선택 후 '제출'.

# 5. 엑셀 파싱 구현 방식 (qbr과 동일)
- **FileReader API**로 `.xlsx` 파일을 바이트로 읽음.
- **SheetJS (CDN)**로 2차원 배열(표)로 변환.
- 파싱 흐름: 파일 선택 → FileReader 읽기 → SheetJS 변환 → JavaScript 배열로 Firebase 업로드.
- 업로드 위치: **Setting View**.

---

# 6. 기타 필수 구현
- **사운드 효과:** 정답, 오답, 카운트다운, **카드 뒤집기** 사운드를 모든 화면에 적용.
- **음소거 토글 버튼:** 모든 화면에 배치.
- **시상식:** '게임 끝내기' 클릭 시 축포 애니메이션 + 효과음 + **최종 누적 점수 순위 표시** (qbr은 순위 표시 없음이었으나 qbc는 표시).
- **정답 공개 없음:** 교사가 구두로 정답을 설명. 시스템에서 정답을 공개하지 않음.
- **문제 진행:** 카드 선택이 끝나도 자동 진행하지 않고, **관리자가 '다음 문제' 버튼을 눌러야 진행** (qbr 동일).
- **복구 로직 없음:** 접속이 끊기면 재접속 후 방 코드와 닉네임을 다시 입력하여 입장.

# 7. DB 구조 (qbc용으로 수정)

```
rooms/
  AB3K/                              ← 방 코드 (알파벳+숫자 4자리)
    │
    ├─ meta/
    │    status: "playing"           ← playing | ended
    │    createdAt: 1717123400000
    │                                   (qbr의 playerCount 제거 — 플레이어 수 선택 없음)
    │
    ├─ gameState/
    │    currentQuestion: 2          ← 현재 문제 인덱스 (0부터 시작)
    │    phase: "answering"          ← idle | countdown | answering | carding | ended
    │                                   (carding: 정답자 카드 선택 단계)
    │    countdownStartAt: 1717123456789
    │    questionStartAt:  1717123462000
    │    ultraArmed: false           ← 관리자가 '초고위험카드' 버튼을 눌러 예약한 상태
    │                                   (카운트다운 시작 시 카드 배분에 반영 후 자동 false로 리셋)
    │    explosionEnabled: true      ← 관리자가 대폭발 포함 여부를 켜고 끔
    │
    ├─ questions/
    │    0/
    │      type: "객관식"
    │      content: "문제 텍스트"
    │      imageUrl: ""
    │      answer: "사과"
    │      choices: ["사과","바나나","포도","딸기"]
    │                                   (qbr의 randomKeys 제거 — 객관식 난수 미사용)
    │    1/
    │      type: "단답형"
    │      content: "..."
    │      answer: "정답1|정답2"
    │      imageUrl: ""
    │
    ├─ players/
    │    <nickname 또는 uid>/
    │      nickname: "홍길동"          ← board 표시용
    │      connected: true
    │      totalScore: 35             ← 누적 점수 (게임 끝까지 비공개, 종료 시 순위 산출)
    │
    ├─ cards/
    │    2/                           ← 문제 인덱스별 카드 세트 (총 28장)
    │      ultraMode: false           ← 이 문제에 초고위험카드가 포함됐는지
    │      explosionTriggeredBy: null ← 대폭발을 발동시킨 플레이어 닉네임
    │      deck/
    │        0/  type: "normal"    score: 7    takenBy: null
    │        1/  type: "risk"      score: 11   takenBy: "홍길동"
    │        2/  type: "highRisk"  score: -20  takenBy: null
    │        3/  type: "double"    score: 0    takenBy: null   ← 세트당 최대 1장, 자체 점수 0
    │        4/  type: "ultra"     score: -40  takenBy: null   ← ultraMode일 때만
    │        5/  type: "explosion" score: 0    takenBy: null   ← ultraMode일 때만, 대폭발
    │        ...
    │        27/ type: "normal"    score: 5    takenBy: null
    │
    │      (type 값: normal(노랑,19) | risk(주황,5) | highRisk(빨강,4 또는 3) | ultra(검정,0~1))
    │      (double·explosion은 각각 highRisk·ultra 풀에서 뽑히지만, 기존 코드 관례를 유지해
    │       tier가 아닌 별도 type("double"/"explosion")으로 구분하고 score는 0으로 고정한다.
    │       — 2배카드·대폭발 판정 로직이 score 문자열이 아닌 type 값을 기준으로 동작하기 때문)
    │      (takenBy에 닉네임이 기록되면 소진 = 뒤집힘)
    │      (트랜잭션으로 takenBy가 null일 때만 기록 성공)
    │

           홍길동/

             correct: true

             gainedScore: -20         ← 이번 문제 획득 점수 (2배 반영 후)
           김철수/
             value: "바나나"
             correct: false
             submittedAt: 1717123461000
             pickedCards: []          ← 오답이라 카드 미선택
```

## DB 설계 메모
- **카드 소진 처리:** `cards/N/deck/M/takenBy`를 Firebase **트랜잭션**으로 갱신. 값이 `null`일 때만 자신의 닉네임을 기록해 성공, 이미 값이 있으면 실패 → 클라이언트는 다른 카드 재선택.
- **2배카드:** `pickedCards`에 카드 인덱스 2개가 쌓임. 두 번째 카드 점수 × 2를 `gainedScore`에 반영.
- **누적 점수:** 문제 종료 시 `gainedScore`를 `players/<id>/totalScore`에 더함. board에는 노출하지 않고, `meta/status`가 `ended`가 될 때만 순위 계산·표시.
- **카드 배분 저장 시점:** 관리자가 카운트다운을 시작할 때 `cards/N/deck`에 28장을 생성·저장(2배카드 1장 제한 로직 포함).
